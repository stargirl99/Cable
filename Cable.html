08

UNDO / RESTORE

After a move-based One-Time Sort, Cable writes a sort_log.json in the sorted folder so you can fully reverse the operation.

1

RUN CABLE AGAIN ON THE SAME FOLDER

Cable detects sort_log.json and asks if you want to undo.

2

CONFIRM RESTORE

All files are moved back to their original locations in reverse order. Empty subfolders are left behind.

3

LOG IS DELETED

Once restore completes, sort_log.json is removed so the operation cannot be undone twice.

‚ÑπÔ∏è

Undo is only available for move operations, not copy operations.

09

DUPLICATE DETECTION

Built-in protection against accidental data loss from file name collisions.

Scenario What Cable does
Same name, same content Source file is skipped. Counter shown in final summary.
Same name, different content Destination renamed to filename (1).ext, filename (2).ext, etc. Both files are kept.

Hash comparison uses SHA-256 computed in 64 KB chunks for speed on large files.

CONFIGURATION

Cable uses two config files: cable.json (structure & defaults) and ~/.sortconfig.toml (session state).

// cable.json ‚Äî FULL CUSTOMIZATION

Lives next to cable.py. Auto-created on first run. Edit freely to change category names, subfolder paths, icons, colors, which extensions go where, OS default paths, date-media categories, and session defaults. Add entirely new categories by appending entries to the categories array.

// cable.json ‚Äî add a custom category
"categories": [{ "key": "üéÆ Games", "folder": "Games",
  "icon": "üéÆ", "color": "#a78bfa", "extensions": [".rom", ".nds"] }]
# ~/.sortconfig.toml ‚Äî session state (auto-saved after each run)
run_mode = "1" # 1 = Once, 2 = Watcher
sort_mode = "alpha" # alpha | ext | size | date | combo
copy_mode = false # true = copy, false = move
date_subfolders = false # YYYY/MM sub-subfolders for media
dest_mode = "here" # here | defaults | where
exclude_patterns = ["desktop.ini", "*.tmp", "Thumbs.db"]
